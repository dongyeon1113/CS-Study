# Chapter 4. 병행 제어 (Concurrency Control) - I

## 1. 데이터의 접근 (Data Access)
- 컴퓨터 시스템에서 데이터가 이동하는 방식: 
    1. 데이터가 저장된 위치(Memory)에서 읽어옴.
    2. 연산하는 위치(CPU)에서 결과를 계산.
    3. 다시 저장된 위치로 결과를 씀.
- **문제 발생**: 여러 주체(Process/Thread)가 하나의 데이터를 동시에 공유하며 연산할 때 데이터의 불일치(Inconsistency)가 발생할 수 있음.

---


## 2. Race Condition (경쟁 상태)
- **정의**: 여러 프로세스가 동시에 공유 데이터에 접근하여 조작할 때, 실행 순서에 따라 결과가 달라지는 상태.
- **OS에서 Race Condition이 발생하는 상황**:
    1. **Kernel 모드 수행 중 인터럽트 발생**: 커널 변수를 수정하던 중 인터럽트가 발생하여 다른 루틴이 동일 변수를 수정하는 경우.
    - 해결책: 변수를 kernel모드에서 수정하는 동안에는 인터럽트를 못하도록함.
    2. **Process가 System Call을 하여 Kernel 모드로 수행 중일 때 Context Switch 발생**: 커널 내부 데이터를 수정하던 중 다른 프로세스에게 CPU가 넘어가는 경우.
    - 해결책: 커널 모드에서 수행 중일 때는 CPU를 preempt하지 않음. 커널 모드에서 사용자 모드로 돌아갈 때 preempt
    3. **Multiprocessor 환경**: 여러 CPU가 공유 메모리 내의 커널 데이터에 동시 접근하는 경우.
    - 해결책: 한번에 하나의 CPU만이 커널에 들어갈 수 있게 하는 방법, 커널 내부에 있는 각 공유 데이터에 접근할 때마다 그 데이터에 대한 lock/unlock을 하는 방법

## 3. Critical Section (임계 구역)
- **정의**: 공유 데이터에 접근하는 코드 영역.
- **해결 목표**: 한 프로세스가 임계 구역에 있으면 다른 프로세스는 들어오지 못하게 해야 함.

### 임계 구역 문제 해결을 위한 3가지 충족 조건
1. **Mutual Exclusion (상호 배제)**: 프로세스 P1이 임계 구역에서 실행 중이면 다른 프로세스들은 그 구역에 들어갈 수 없음.
2. **Progress (진행)**: 임계 구역에 아무도 없고 들어가고자 하는 프로세스가 있다면 바로 들여보내 주어야 함 (Deadlock 방지).
3. **Bounded Waiting (유한 대기)**: 프로세스가 임계 구역에 들어가려고 요청한 후부터 허용될 때까지, 다른 프로세스들이 임계 구역에 들어가는 횟수에 한계가 있어야 함 (Starvation 방지).

---

## 4. SW적인 해결 알고리즘 (Peterson's Algorithm)
- **개념**: `turn`(누구 차례인지)과 `flag`(의사가 있는지) 변수를 사용하여 두 프로세스 간의 임계 구역 문제를 해결.
- **한계**: **Busy Waiting (Spin lock)** 문제 발생. CPU를 잡고 있으면서도 조건을 만족하지 못해 계속 loop를 돌며 대기하여 CPU 자원을 낭비함.

---

## 5. HW적인 해결 방법 (Synchronization Hardware)
- **문제의 근본 원인**: 데이터를 읽고 쓰는 작업이 원자적(Atomic)으로 이루어지지 않기 때문.
- **해결**: `Test_and_Set`과 같은 하드웨어 명령어를 사용하여 **읽기와 쓰기를 한 번의 Instruction으로 처리**함으로써 원자성을 보장함.

---

## 6. Semaphores (세마포어)
- **정의**: 앞서 언급한 방식들을 추상화시킨 자료구조. 일종의 추상 자료형으로 정수값(S)과 두 가지 연산(`P`, `V`)으로 구성됨.
- **연산 종류**:
    - **P(S)**: 세마포어 값을 감소시킴 (자원을 획득하는 과정, S가 0 이하이면 대기).
    - **V(S)**: 세마포어 값을 증가시킴 (자원을 반납하는 과정).
- **type**: Counting semaphore(도메인이 0이상인 임의의 정수값 자원의 개수를 세야할때), Binary semaphore(0또는 1값만 가질 수 있는 semaphore 주로 mutual exclusion(lock/unlock)에 사용)

### 7. Block & Wakeup 방식 (Sleep lock)
- Busy Waiting의 단점을 해결하기 위해, 자원을 얻지 못한 프로세스를 Ready Queue가 아닌 **Semaphore Queue(Wait Queue)**에 넣고 잠들게 함.
- 자원이 반납되면 잠들어 있는 프로세스를 깨워 실행시킴.
