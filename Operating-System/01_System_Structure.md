# Chapter 1. 컴퓨터 시스템의 구조

## 1. 컴퓨터 시스템 구조 (하드웨어 관점)

| 구성 요소 | 설명 |
| :--- | :--- |
| **CPU** | 매 클럭(Clock)마다 메모리에서 인스트럭션을 읽어 실행 |
| **Memory** | CPU의 작업 공간 |
| **Disk** | 파일 시스템 역할 및 메모리 부족 시 Swap Area로 활용 |
| **Device Controller** | 각 I/O 장치를 관리하는 전담 CPU (하드웨어) |
| **Device Driver** | CPU가 수행하는 코드로, Controller에게 요청을 전달하는 인터페이스 (소프트웨어) |
| **Local Buffer** | Device Controller의 전용 작업 메모리 |
| **DMA Controller** | CPU 대신 장치와 메모리 사이의 데이터 복사를 전담하여 오버헤드 감소 |
| **Timer** | CPU 독점 방지를 위해 일정 시간 뒤 OS에게 제어권이 넘어가도록 인터럽트 발생 |

---

## 2. CPU 내의 특수 장치

* **Registers**: 메모리보다 빠른 저장소. 특히 **PC(Program Counter)**는 다음에 실행할 메모리 주소를 가리킴.
* **Mode Bit**: CPU 내 명령 실행 권한을 표시.
  * **0 (Kernel Mode)**: OS 수행 중. 모든 특권 명령 가능.
  * **1 (User Mode)**: 사용자 프로그램 수행 중. I/O 등 직접 접근 불가.
* **Interrupt Line**: CPU가 인스트럭션 하나를 끝낼 때마다 체크하는 신호선.

---

## 3. 인터럽트 (Interrupt) 메커니즘

### 개념
CPU가 프로그램을 실행 중일 때, 주변 장치나 예외 상황 발생 시 이를 처리하기 위해 보내는 신호. **현대의 운영체제는 인터럽트에 의해 구동됨.**

### 과정
1. 인터럽트 발생
2. CPU가 현재 상태 저장
3. 제어권이 OS로 이동 (**Mode Bit가 0으로 변경**)
4. **인터럽트 벡터**를 통해 **서비스 루틴(ISR)** 실행

### 종류
1. **Hardware Interrupt**: 하드웨어 장치가 발생시킴 (Timer, I/O 완료 등)
2. **Software Interrupt (Trap)**:
   * **Exception**: 프로그램 오류 (0으로 나누기, 메모리 권한 위반 등)
   * **System Call**: 사용자 프로그램이 OS 서비스를 받기 위해 커널 함수를 호출하는 의도적 호출

---

## 4. 인터럽트 관련 용어
* **인터럽트 벡터**: 해당 인터럽트의 처리 루틴 주소를 가지고 있는 테이블.
* **인터럽트 처리 루틴 (ISR)**: 해당 인터럽트를 실제로 처리하는 커널 함수.

---

## 5. 입출력(I/O)의 수행 과정

I/O는 보안상 **특권 명령**이므로 사용자 프로그램이 직접 수행할 수 없음.

1. 사용자 프로그램이 **System Call**(사용자 프로그램이 OS 서비스를 받기 위해 커널 함수를 호출하는 것)을 통해 OS에 요청.
2. CPU 제어권이 OS로 넘어가며 Mode Bit가 0으로 바뀜.
3. OS가 해당 장치 컨트롤러에게 I/O를 명령.
4. 장치 컨트롤러가 데이터를 Local Buffer에 다 채우면 **Interrupt** 발생.
5. OS가 읽은 데이터를 사용자 프로그램의 메모리 영역에 복사해줌.

### 동기식 vs 비동기식 I/O
* **동기식 (Synchronous I/O)**: I/O 요청 후 결과가 올 때까지 기다림.
  * **방법 1**: CPU를 낭비하며 대기 (매 시점 하나의 I/O만 수행).
  * **방법 2**: I/O 완료 시까지 해당 프로그램의 CPU를 박탈하고 다른 프로그램에게 제어권 이양.
* **비동기식 (Asynchronous I/O)**: I/O 요청만 하고 결과와 관계없이 바로 다음 동작 수행.

---

## 6. DMA (Direct Memory Access)

* **배경**: 잦은 I/O 인터럽트는 CPU의 효율을 저하시킴.
* **역할**: 장치 버퍼의 데이터를 메모리에 바이트(Byte) 단위가 아닌 **블록(Block) 단위**로 직접 복사.
* **장점**: 복사 완료 시 인터럽트를 딱 한 번만 발생시켜 CPU 오버헤드 최소화.
